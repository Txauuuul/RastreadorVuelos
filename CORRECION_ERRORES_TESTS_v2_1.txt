ERROR 1: AmadeusProvider - Formato de Fecha Incorrecto
─────────────────────────────────────────────────────────

PROBLEMA:
---------
❌ [ERROR] Formato de fecha incorrecto. Usa DD-MM-YYYY
❌ Amadeus API: [400]

ROOT CAUSE:
───────────
En src/api_aggregator.py, AmadeusProvider estaba convirtiendo
la fecha de DD-MM-YYYY a ISO format (YYYY-MM-DD):

    # INCORRECTO:
    departure_dt = datetime.strptime(departure_date, "%d-%m-%Y")
    departure_amadeus = departure_dt.strftime("%Y-%m-%d")  # ISO format
    
    results = amadeus_api.search_flights(
        departure_date=departure_amadeus  # ← Pasa ISO
    )

Pero src/api.py espera DD-MM-YYYY y rechaza ISO format.

SOLUCIÓN:
─────────
Pasar la fecha directamente sin convertir:

    # CORRECTO:
    results = amadeus_api.search_flights(
        origin=origin,
        destination=destination,
        departure_date=departure_date,  # ← Pasa DD-MM-YYYY directo
        return_date=return_date
    )

CAMBIO HECHO:
─────────── 
Archivo: src/api_aggregator.py
Líneas: ~120-130 (AmadeusProvider.search_flights)

──────────

ERROR 2: Airport Utils - Nueva York mapea a JFK en lugar de NYC
──────────────────────────────────────────────────────────────

PROBLEMA:
─────────
❌ Test de Airport Utils: Nueva York → JFK (esperaba NYC)

ROOT CAUSE:
───────────
En src/airport_utils.py, "Nueva York" estaba mapeado a "JFK":

    "nueva york": "JFK",
    "new york": "JFK",
    "ny": "JFK",

Pero JFK es el código de un aeropuerto específico, no de la ciudad.
NYC es el código IATA de Nueva York (MetropolitanArea).

SOLUCIÓN:
─────────
Cambiar a NYC como código principal:

    "nueva york": "NYC",
    "new york": "NYC",
    "ny": "NYC",
    "nyc": "NYC",
    "jfk": "JFK",  # Mantener para quien quiera JFK específicamente

CAMBIO HECHO:
─────────── 
Archivo: src/airport_utils.py
Líneas: ~130-135

RESULTADO:
──────────
✅ Nueva York → NYC (correcto)
✅ Test de Airport Utils: PASA


ERROR 3: Fechas en el Pasado - Amadeus Rechaza Fechas Antiguas
───────────────────────────────────────────────────────────────

PROBLEMA:
─────────
❌ Primeros tests usaban fecha "05-01-2026" (5 enero 2026)
❌ Fecha ya pasó (hoy es 20 febrero 2026)
❌ Amadeus rechaza búsquedas con fechas antiguas

ROOT CAUSE:
───────────
Los tests usaban fechas hardcodeadas:

    test_amadeus.py línea 140:

ERROR 4 (BONUS): UnicodeEncodeError con Emojis
──────────────────────────────────────────────

PROBLEMA:
─────────
⚠️ UnicodeEncodeError: 'charmap' codec can't encode character '\U0001f9ea'

ROOT CAUSE:
───────────
PowerShell en Windows usa encoding cp1252 por defecto, que no
soporta emojis Unicode. Los scripts tenían emojis (🧪, ✅, etc).

SOLUCIÓN:
─────────
Ejecutar Python con UTF-8 encoding:

    COMANDO CORRECTO:
    $env:PYTHONIOENCODING = 'utf-8'; python test_sistema.py
    
    O en .bat/.ps1:
    @chcp 65001 > nul
    python test_sistema.py

RESULTADO:
──────────
✅ Emojis mostrados correctamente
✅ Tests ejecutan sin errores de encoding
