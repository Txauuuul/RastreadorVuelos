✅ BENEFICIOS CONCRETOS
═══════════════════════════════════════════════════════════════════════

1. RESILIENCIA MEJORADA
   ──────────────────

   Escenario: API_X cae (error 500)
   
   ANTES: Bot crashea con ImportError o exception
   AHORA: APIAggregator ignora API_X, continúa con otras
   
   Ejemplo log:
   ✓ Amadeus: 150€ encontrado
   ✗ FlyScraper: [500] Server error
   ✓ GoogleFlights: 140€ encontrado
   → Retorna: 140€ (mejor de lo que funcionó)


2. PRECIOS MÁS BARATOS
   ─────────────────

   Escenario: Búsqueda Madrid → Barcelona
   
   ANTES: Amadeus retorna 200€
          (usuario solo ve eso)
   
   AHORA: Amadeus 200€
          FlyScraper 160€
          GoogleFlights 180€
          → Retorna: 160€ (40€ de ahorro)


3. VELOCIDAD MEJORADA
   ─────────────────

   Tiempo de búsqueda:
   
   ANTES (secuencial):
   Amadeus: 8s
   FlyScraper: 10s
   GoogleFlights: 9s
   TOTAL: 27 segundos
   
   AHORA (paralelo con threading):
   Max(8, 10, 9) = 10 segundos
   TOTAL: 10 segundos (2.7x más rápido)


4. EXTENSIBILIDAD
   ──────────────

   Agregar nueva API es trivial:
   
   class TuAPIProvider(APIProvider):
       def __init__(self):
           super().__init__("TuAPI")
       
       def search_flights(self, origin, dest, dates):
           ...
   
   agregador.add_provider("TuAPI", TuAPIProvider())
   
   ¡Automáticamente incluida en futuras búsquedas!

⚠️ CONSIDERACIONES
═══════════════════════════════════════════════════════════════════════

1. CUOTA DE APIs
   ──────────────
   
   Amadeus: Pagada (tienes créditos limitados)
   FlyScraper: Gratuita (pero con límites mensuales)
   GoogleFlights: Gratuita (pero con límites mensuales)
   
   → Si usas las 3 en paralelo, consumirás cuota 3x más rápido

2. THREADING vs ASYNCIO
   ─────────────────────
   
   Usamos threading porque:
   ✓ requests library no es async-ready
   ✓ Threading es más simple de entender
   ✗ No es perfecto (GIL de Python), pero suficiente
   
   Alternativa: Podrías usar asyncio + aiohttp si quieres futuro

3. TIMEOUT
   ───────
   
   Cada API tiene timeout de 10 segundos
   Total de búsqueda: máximo 15 segundos
   
   Si una API tarda más, se salta (no bloquea otras)

4. ORDEN DE RETORNO
   ────────────────
   
   Se retorna primero el mejor precio, luego todas opciones
   Orden de FlightOfferList no garantizado (depende de threading)

❓ PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════

P: ¿Tengo que cambiar main.py?
R: No. run_combined.py ya lo usa automáticamente.

P: ¿Cómo deshabilito una API?
R: agregador.disable_provider("GoogleFlights")

P: ¿Cómo agrego Kayak?
R: Creas KayakProvider(APIProvider) y agregador.add_provider()

P: ¿Qué pasa si TODAS las APIs fallan?
R: mejor_vuelo = None, no se envía alerta, es correcto.

P: ¿Es thread-safe?
R: Sí, cada thread es independiente, GIL Python no es problema aquí.

P: ¿Puedo usar asyncio en lugar de threading?
R: Sí, pero requiere cambiar requests → aiohttp (más trabajo).
