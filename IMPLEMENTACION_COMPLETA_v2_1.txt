💡 CÓMO FUNCIONA EL AGREGADOR
═══════════════════════════════════════════════════════════════════════

CUANDO BUSCAS Un VUELO:
──────────────────────

1. SearchWorker llama: agregador.search("MAD", "BCN", "05-03-2026")

2. APIAggregator EJECUTA EN PARALELO:
   
   ├─ Thread 1: AmadeusProvider
   │  └─ Llamada: amadeus_api.search_flights(...)
   │
   ├─ Thread 2: FlysScraperProvider  
   │  └─ Llamada: requests.get(flyscraper_api, headers=...)
   │
   └─ Thread 3: GoogleFlightsProvider
      └─ Llamada: requests.get(googleflights_api, headers=...)
   
   Todas SIMULTÁNEAMENTE (no una después de otra)

3. Consolida resultados:
   
   [FlightOffer(120€, FlyScraper),
    FlightOffer(130€, Amadeus),
    FlightOffer(140€, GoogleFlights)]

4. Encuentra el mínimo: 120€

5. Retorna:
   (mejor_vuelo=FlightOffer(..., 120€, FlyScraper),
    todos_vuelos=[...])

6. SearchWorker envía alerta Telegram:
   "💰 120€ en FlyScraper (de 3 opciones comparadas)"


GESTIÓN DE FALLOS:
──────────────────

Si FlyScraper falla:

✓ Amadeus: 130€ → OK
✗ FlyScraper: ERROR (timeout, rate limit, etc) → SE IGNORA
✓ GoogleFlights: 140€ → OK

Resultado: Retorna 130€ (mejor de lo que funcionó)
No crashea, No aviso de error, Sistema resiliente ✓

❓ PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════

P: ¿Tengo que cambiar main.py?
R: No. Ya está integrado. run_combined.py y flight_search_worker.py
   automáticamente usan APIAggregator.

P: ¿Cómo agregoplexo una 4ª API?
Ř: Creas una clase que implemente APIProvider y 
   agregador.add_provider("MiAPI", MiAPIProvider())

P: ¿Es thread-safe?
R: Sí. Cada thread es independiente. No hay data races.

P: ¿Cómo deshabilito una API temporalmente?
R: agregador.disable_provider("FlyScraper")

P: ¿Qué pasa si todas las APIs fallan?
R: mejor_vuelo = None, SearchWorker maneja gracefully.

P: ¿Puedo usar esto en Render?
R: Sí, 100%. Solo executa: python run_combined.py

P: ¿Consume más cuota?
R: Sí, 3x más (usas 3 APIs). Pero usualmente más barato.
